# Labeller Validation

**Goal:** Use the EC2 labeller-validation app's collected human ratings as ground truth to measure (and ultimately iterate on) the labeller prompt.

**Status:** Shipped. Hybrid per-type labeller (`classify_scaffolding` + `classify_rapport`) lifts test_v2 kappa **0.725 -> 0.782** without any binary regression.

## Final result (claude-opus-4-6, test_v2, n=147)

Per-type routed labeller: scaffolding rows go to `classify_scaffolding.txt`
(verbatim of the rule-based v2), rapport rows go to `classify_rapport.txt`
(generated by Claude Opus 4.7 1M from a minimal meta-prompt over the 343
train examples).

| Metric | v2 baseline | **Hybrid (shipped)** | Lift |
|---|---:|---:|---:|
| 3-way kappa | 0.725 | **0.782** | **+0.057** |
| 3-way accuracy | 82.3% | **85.7%** | +3.4pp |
| Scaffolding kappa | 0.783 | **0.783** | flat |
| Rapport kappa | 0.664 | **0.775** | **+0.111** |
| Binary kappa | 0.796 | **0.796** | flat |
| Partial recall | 51% | **73%** | +22pp |

- Polar agreement preserved: 1 eff→ineff swap across 110 polar items (99.1%).
- Partial recall up to 73% (from 51%) without sacrificing polar precision.
- Rapport kappa lifts by +0.111 on the type where v2 underperformed.

## What got us here

Tried four design routes after the v2 baseline:

| Prompt | Approach | Test kappa | Notes |
|---|---|---:|---|
| v2 | Rule-based (existing) | 0.725 | Baseline |
| v3 | v2 + rule refinements | regressed | Rules can't disambiguate substantive vs stylistic |
| v4 | v2 + 4 hand-picked examples | 0.741 | Asymmetric: rapport +0.110, scaffolding -0.083 |
| v5 | Meta-prompt with priming (343 train ex) | 0.739 | Type-conditional rules, primed by my hypothesis |
| v6 | Minimal meta-prompt (no priming) | 0.741 | Trigger-phrase rules, found asymmetry on its own |
| **hybrid** | v2 for scaffolding + v6 for rapport | **0.782** | Routes per annotation_type |

All single-prompt variants plateau at ~0.74 -- a structural ceiling driven by inter-rater inconsistency on the scaffolding "worked + improvement note" pattern (some reviewers call it effective, some call it partial). Routing by type lets each domain use the prompt its humans actually agree with.

## Context

The labeller (`prompts/annotator/labeller/classify_v2.txt`) takes a SAR (situation/action/result) text and outputs `effective | partial | ineffective`. The validation app at `http://54.184.80.37/labeller-validation/` lets human reviewers do exactly that task by hand on a stratified sample of `step_up_annotations.jsonl`. This produces a labelled dataset of (SAR text, human verdict) pairs that we can measure the labeller against.

Reviewer data is written to S3: `s3://kylel-alexisr-edu/normalized-contains-pii/validation/labeller_validation/{reviewer}.jsonl`.

## Steps

### Step 1: Pull data locally — DONE

Pulled S3 → `data/labeller_validation/`:
- `{dani,nathan,query,rebecca}.jsonl`
- `eval/labeller_split_meta_v1.json` + `labeller_{train,test}_v1.jsonl` (May 7 split, excludes Nathan)
- `step_up_annotations.jsonl` (source SAR text, 19 MB)

### Step 2: Human ceiling — DONE (zero overlap, see above)

`validation/3_labeller_validation.py::overlap_and_ceiling()` computes pairwise Cohen's kappa across reviewers. All pairs returned n=0.

**Follow-up if a ceiling becomes important:** run a side exercise where 2+ reviewers rate a shared set of ~50 keys.

### Step 3: v2 stratified split (Nathan included) — DONE

`validation/3_labeller_validation.py::build_v2_split()` writes:
- `eval/labeller_train_v2.jsonl` (343 rows)
- `eval/labeller_test_v2.jsonl` (147 rows)
- `eval/labeller_split_meta_v2.json` (seed=42, stratified by annotation_type × human_rating)

All six (annotation_type × rating) strata are represented in both train and test. Smallest stratum: scaffolding|partially_effective (34 train / 15 test).

### Step 4: Baseline labeller eval — DONE

`PYTHONPATH=. python validation/4_run_labeller_eval.py --model-profile anthropic --split test_v2`

- Loads test_v2 (147 rows), joins SAR text from `step_up_annotations.jsonl` (all 147 resolved).
- Builds `classify_v2` prompts with `{annotation_type}`, `{situation}`, `{action}`, `{result_text}`.
- Runs Claude Opus 4.6 in batch mode.
- Writes predictions JSONL + metrics JSON to `data/labeller_validation/eval/`.

**Label mapping:** labeller outputs `partial`; human ratings use `partially_effective`. Mapped in `HUMAN_TO_LLM` for comparison.

### Step 5: Iterate (next plan)

After baseline numbers land, the advisor loop pattern from `history/v5_detection_iteration/` is the next move:
- Inspect labeller mis-classifications on the **train** set
- Advisor proposes prompt patches → re-eval on train → repeat
- **Test set is held out** — only run once at the end for an unbiased number

## Open questions

1. **Should we collect overlap ratings?** Without a human ceiling, we can't say whether labeller mistakes are correctable. ~50 shared items across 2 reviewers would give a usable ceiling.
2. **Nathan dominates (60% of data).** Worth checking if his rating distribution is systematically different from dani/rebecca/query before treating the pooled dataset as one source of truth.
3. **dani's 33% skip rate** — what was the skip reason? If skips cluster on a specific annotation pattern, that's a labeller signal we're losing.
